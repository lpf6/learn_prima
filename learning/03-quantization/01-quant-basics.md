# 3.1 量化基础

## 什么是量化？

量化是将连续值映射到离散值的过程。在深度学习中，通常指将 FP32 或 FP16 的权重和激活值转换为更低精度的表示（如 INT8、INT4）。

## 为什么需要量化？

### 1. 内存节省

```
FP32 模型: 7B 参数 × 4 bytes = 28 GB
INT8 模型: 7B 参数 × 1 byte  = 7 GB
INT4 模型: 7B 参数 × 0.5 byte = 3.5 GB
```

### 2. 计算加速

- INT8 运算比 FP32 快 2-4 倍
- INT4 运算更快（如果有硬件支持）
- 减少内存带宽压力

### 3. 能耗降低

低精度运算消耗更少能量，对移动设备和边缘计算尤为重要。

## 量化基本方法

### 1. 线性量化

```
量化: q = round(x / scale) + zero_point
反量化: x = (q - zero_point) × scale
```

### 2. 对称量化

```
量化: q = round(x / scale)
反量化: x = q × scale

特点: zero_point = 0，简化计算
```

### 3. 非对称量化

```
量化: q = round((x - min) / scale)
反量化: x = q × scale + min

特点: 更好地利用量化范围
```

## 块量化

Prima.cpp 使用块量化（Block Quantization）来提高精度：

### 基本思想

将权重分成小块，每块有独立的缩放因子：

```
原始权重: [w0, w1, w2, ..., w31]

块量化后:
块 0: scale_0, [q0, q1, ..., q31]  (32 个量化值共享一个 scale)
块 1: scale_1, [q32, q33, ..., q63]
...
```

### 为什么块量化更好？

1. **局部精度**：相邻权重通常数值范围相近，共享缩放因子更准确
2. **存储效率**：每块只需一个缩放因子
3. **计算效率**：块大小通常是 2 的幂，便于并行计算

## 量化精度评估

### 常用指标

1. **均方误差 (MSE)**
```
MSE = mean((x - x_dequant)²)
```

2. **信噪比 (SNR)**
```
SNR = 10 × log10(var(x) / var(x - x_dequant))
```

3. **困惑度变化**：量化前后模型困惑度的差异

## 量化粒度

| 粒度 | 描述 | 精度 | 存储 |
|------|------|------|------|
| **逐张量** | 整个张量一个 scale | 低 | 最少 |
| **逐通道** | 每个通道一个 scale | 中 | 中等 |
| **逐块** | 每个块一个 scale | 高 | 较多 |
| **逐组** | 每组元素一个 scale | 最高 | 最多 |

## Prima.cpp 中的块大小

// 文件：ggml/src/ggml-common.h

## 量化流程

```
原始模型 (FP32/FP16)
       │
       ▼
┌──────────────────┐
│   校准数据准备    │  选择代表性输入数据
└──────────────────┘
       │
       ▼
┌──────────────────┐
│   量化范围确定    │  计算每块的最大/最小值
└──────────────────┘
       │
       ▼
┌──────────────────┐
│   缩放因子计算    │  scale = (max - min) / (2^bits - 1)
└──────────────────┘
       │
       ▼
┌──────────────────┐
│   权重量化        │  q = round(w / scale)
└──────────────────┘
       │
       ▼
量化模型 (INT4/INT8)
```

## 反量化过程

推理时需要将量化值反量化：

```cpp
// 简化的反量化示例
float dequantize_q4_0(block_q4_0 block, int i) {
    // block.d: 缩放因子 (FP16)
    // block.qs[i]: 量化值 (4-bit，存储在 8-bit 中)

    // 将 4-bit 值转换为有符号整数 [-8, 7]
    int8_t q = (block.qs[i] & 0x0F) - 8;

    // 反量化
    return q * block.d;
}
```

## 量化对模型性能的影响

### 精度损失

| 量化类型 | 典型困惑度增加 | 适用场景 |
|----------|---------------|----------|
| FP16 | ~0% | 基准 |
| Q8_0 | ~0.1% | 高精度需求 |
| Q6_K | ~0.3% | 平衡精度和速度 |
| Q5_0/Q5_1 | ~0.5% | 通用场景 |
| Q4_0/Q4_1 | ~1-2% | 内存受限 |
| Q3_K | ~2-3% | 极端内存受限 |
| Q2_K | ~5%+ | 实验性 |

### 推理速度

| 量化类型 | 相对速度 (vs FP16) |
|----------|-------------------|
| Q8_0 | 1.5-2x |
| Q4_0 | 2-3x |
| Q4_K | 2-3x |
