# 实践项目模块

## 概述

本模块提供端到端的实践项目，帮助你综合运用所学知识，完成从理论到实践的转化。

## 项目列表

### 1. [构建推理服务](01-inference-service.md)

**目标**：构建一个生产级的 LLM 推理服务

**你将学到**：
- 服务架构设计
- 请求调度和批处理
- 性能监控和日志
- 负载均衡

**预计时间**：2-3 天

**难度**：⭐⭐⭐⭐

**前置知识**：
- CUDA 基础
- LLM 架构理解
- 后端兼容性

**能力目标**：

| 能做什么 | 还不能做什么 |
|----------|--------------|
| 构建推理服务 API | 处理超大规模并发 |
| 实现动态批处理 | 分布式部署 |
| 监控性能指标 | 自动扩缩容 |

### 2. [自定义内核开发](02-custom-kernel.md)

**目标**：开发一个自定义的 CUDA 内核并集成到项目中

**你将学到**：
- 内核需求分析
- 内核设计和实现
- 性能优化技巧
- 集成测试

**预计时间**：3-5 天

**难度**：⭐⭐⭐⭐⭐

**前置知识**：
- CUDA 编程基础
- GPU 架构理解
- 性能分析工具

### 3. [模型量化实践](03-model-quantization.md)

**目标**：完成一个完整模型的量化流程

**你将学到**：
- 量化策略选择
- 校准数据集准备
- 精度评估方法
- 性能测试对比

**预计时间**：2-3 天

**难度**：⭐⭐⭐⭐

**前置知识**：
- 量化技术基础
- 深度学习基础

### 4. [性能基准测试](04-performance-benchmark.md)

**目标**：建立完整的性能基准测试体系

**你将学到**：
- 测试方法论
- 基准测试套件设计
- 结果分析和可视化
- 优化建议生成

**预计时间**：2-3 天

**难度**：⭐⭐⭐

**前置知识**：
- 性能分析工具
- 统计学基础

## 项目选择建议

```
根据你的目标选择项目：

想构建生产系统？
→ 项目 1：构建推理服务

想深入理解内核优化？
→ 项目 2：自定义内核开发

想掌握量化技术？
→ 项目 3：模型量化实践

想系统评估性能？
→ 项目 4：性能基准测试

时间有限？
→ 项目 4（最全面，涵盖所有知识点）

想挑战自己？
→ 按顺序完成所有项目
```

## 项目完成检查清单

完成每个项目后，检查你是否能够：

### 构建推理服务
- [ ] 实现 RESTful API
- [ ] 支持并发请求处理
- [ ] 实现请求队列管理
- [ ] 添加性能监控
- [ ] 编写服务文档

### 自定义内核开发
- [ ] 完成内核设计文档
- [ ] 实现内核代码
- [ ] 性能达到预期目标
- [ ] 通过单元测试
- [ ] 集成到项目中

### 模型量化实践
- [ ] 选择合适量化策略
- [ ] 准备校准数据集
- [ ] 完成量化流程
- [ ] 精度损失在可接受范围
- [ ] 性能提升明显

### 性能基准测试
- [ ] 设计测试用例
- [ ] 收集性能数据
- [ ] 生成对比报告
- [ ] 分析性能瓶颈
- [ ] 提出优化建议

## 项目评估标准

每个项目将根据以下标准评估：

| 标准 | 权重 | 说明 |
|------|------|------|
| **功能完整性** | 30% | 是否实现所有必需功能 |
| **性能表现** | 25% | 性能是否达到预期 |
| **代码质量** | 20% | 代码是否规范、可维护 |
| **文档完整** | 15% | 文档是否清晰完整 |
| **创新性** | 10% | 是否有创新优化 |
