## 知识点交叉引用

### CUDA 与 CPU SIMD 的对应关系

| CUDA 概念 | CPU SIMD 对应 | 说明 |
|-----------|--------------|------|
| Warp (32线程) | SIMD 向量 | 并行处理单元 |
| Shared Memory | L1 Cache | 快速暂存区 |
| Warp 归约 | 水平操作 | 向量内归约 |
| Shuffle | 重排指令 | 数据交换 |
| __syncthreads() | 内存屏障 | 同步点 |

## 常见问题与知识点关联

### Q1: 为什么量化后推理更快？

```
量化技术 ──► 内存带宽减少 ──► 数据传输加速
     │
     └──► INT8 运算 ──► dp4a 指令 ──► 计算加速
              │
              └──► Tensor Core ──► 矩阵乘加速
```

### Q2: Flash Attention 为什么省内存？

```
Flash Attention
     │
     ├──► 分块计算 ──► 不存储完整注意力矩阵
     │
     └──► 在线 Softmax ──► 增量更新结果
              │
              └──► 内存复杂度: O(N) vs O(N²)
```

### Q3: 如何支持新 GPU 架构？

```
新架构支持
     │
     ├──► 计算能力检测 ──► ggml_cuda_info()
     │
     ├──► 条件编译 ──► #if __CUDA_ARCH__ >= XXX
     │
     └──► 内核优化 ──► 使用新特性指令
```

## 总结

理解知识点之间的关联可以帮助你：

1. **建立知识体系**：不是孤立地学习每个概念
2. **理解设计决策**：为什么代码这样写
3. **快速定位问题**：遇到问题时知道涉及哪些知识点
4. **举一反三**：将学到的知识应用到新场景
